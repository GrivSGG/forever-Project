# Проверка подписки в реальном времени

## Что было сделано

Реализована система проверки подписки каждую секунду во время работы игры. Если подписка становится неактивной (истекла или была отозвана), игра автоматически завершается.

## Изменения в API (forever-auth-api/server.js)

### Новый endpoint: `/api/auth/verify-subscription`

Быстрая проверка подписки для периодических запросов из loader.

**Запрос:**
```json
{
  "username": "имя_пользователя"
}
```

**Ответ (успех):**
```json
{
  "success": true,
  "active": true,
  "type": "30days",
  "expiresAt": "2024-12-31T23:59:59.000Z",
  "betaAccess": false
}
```

**Ответ (подписка неактивна):**
```json
{
  "success": false,
  "error": "NO_SUBSCRIPTION",
  "active": false,
  "message": "No active subscription"
}
```

**Возможные ошибки:**
- `USER_NOT_FOUND` - пользователь не найден
- `BANNED` - аккаунт заблокирован
- `NO_SUBSCRIPTION` - нет активной подписки
- `SUBSCRIPTION_EXPIRED` - подписка истекла (автоматически деактивируется)

## Изменения в Loader (MegaSelfCodeLoader.cpp)

### 1. Новая глобальная переменная
```cpp
std::atomic<bool> g_SubscriptionCheckRunning{ false };
```

### 2. Новая функция: `VerifySubscription()`
Быстрая проверка подписки с таймаутом 5 секунд.

```cpp
bool VerifySubscription(const std::wstring& username)
```

### 3. Новый поток: `SubscriptionCheckThread()`
Проверяет подписку каждую секунду:
- Вызывает `VerifySubscription()` каждую секунду
- Если подписка неактивна - вызывает `TerminateGame()`
- Показывает сообщение об ошибке пользователю
- Автоматически останавливается при завершении игры

### 4. Запуск проверки
Поток запускается сразу после успешного запуска игры:
```cpp
std::thread(SubscriptionCheckThread).detach();
```

### 5. Остановка проверки
Проверка автоматически останавливается:
- При завершении игры (`MonitorProcessThread`)
- При принудительном завершении (`TerminateGame`)

## Как это работает

1. **Пользователь входит в аккаунт** → Loader проверяет подписку через `/api/auth/check`
2. **Игра запускается** → Loader запускает поток `SubscriptionCheckThread`
3. **Каждую секунду** → Loader проверяет подписку через `/api/auth/verify-subscription`
4. **Если подписка неактивна** → Loader завершает игру и показывает сообщение
5. **Игра завершается** → Поток проверки автоматически останавливается

## Сценарии использования

### Сценарий 1: Истечение подписки
1. Пользователь играет с подпиской "7days"
2. Подписка истекает через 7 дней
3. При следующей проверке (в течение 1 секунды) loader обнаруживает истечение
4. Игра автоматически завершается с сообщением "Подписка истекла"

### Сценарий 2: Отзыв подписки helper'ом
1. Пользователь играет с активной подпиской
2. Helper отзывает подписку через панель
3. В течение 1 секунды loader обнаруживает отзыв
4. Игра автоматически завершается с сообщением "Подписка была отозвана"

### Сценарий 3: Бан пользователя
1. Пользователь играет
2. Admin банит пользователя
3. В течение 1 секунды loader обнаруживает бан
4. Игра автоматически завершается

## Производительность

- **Частота проверки:** 1 секунда
- **Таймаут запроса:** 5 секунд
- **Размер запроса:** ~50 байт
- **Размер ответа:** ~150 байт
- **Нагрузка на сеть:** минимальная (~200 байт/сек)

## Безопасность

1. **Проверка на стороне сервера** - loader не может обойти проверку
2. **Автоматическая деактивация** - истекшие подписки деактивируются автоматически
3. **Быстрая реакция** - максимум 1 секунда задержки
4. **Защита от обхода** - даже если пользователь модифицирует loader, сервер не даст доступ

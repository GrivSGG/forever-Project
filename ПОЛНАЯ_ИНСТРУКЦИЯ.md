## 🚀 Как использовать:

### Для администратора:

1. **Вход в админ-панель:**
   ```
   URL: https://GrivSGG.github.io/forever-Project/admin/dashboard.html
   Логин: admin
   Пароль: Su6-N77-B6e-nWj
   ```

2. **Генерация ключей:**
   - Зайти в раздел "Генератор ключей"
   - Выбрать тип лицензии
   - Указать количество (1-100)
   - Нажать "Сгенерировать"
   - Скопировать ключи и отправить покупателю

3. **Управление пользователями:**
   - Просмотр всех зарегистрированных пользователей
   - Просмотр информации о пользователе
   - Удаление пользователей

4. **Просмотр статистики:**
   - Общее количество пользователей
   - Активные лицензии
   - Доход за месяц
   - Продажи за день

### Для пользователей:

1. **Регистрация:**
   - Нажать "Личный кабинет" на главной
   - Заполнить форму регистрации
   - Подтвердить email (опционально)

2. **Покупка лицензии:**
   - Выбрать тариф на странице "Купить"
   - Связаться с поддержкой (Discord/Telegram)
   - Оплатить и получить ключ

3. **Активация:**
   - Войти в личный кабинет
   - Нажать "Активировать ключ"
   - Ввести полученный ключ
   - Ключ привязывается к HWID компьютера

4. **Скачивание:**
   - После активации доступна кнопка "Скачать клиент"
   - Загрузить Forever Client
   - Запустить и играть

## 🔒 Безопасность:

### Реализованная защита:

1. **От брутфорса:**
   - Максимум 5 попыток входа
   - Блокировка на 15 минут
   - Отслеживание по Device ID

2. **От взлома:**
   - Отключение DevTools
   - Защита консоли
   - Мониторинг подозрительной активности
   - Шифрование данных админа

3. **От DDoS:**
   - Rate limiting (встроенный в браузер)
   - Cloudflare (при использовании)
   - Блокировка подозрительных IP

4. **Защита данных:**
   - Хеширование паролей
   - Шифрование админ-данных
   - Безопасное хранение в localStorage
   - Проверка сессий

### Рекомендации:

1. **Для GitHub Pages:**
   - Включить HTTPS (автоматически)
   - Добавить Cloudflare для DDoS защиты
   - Использовать внешний backend для критичных данных

2. **Для production:**
   - Перенести на реальный сервер
   - Использовать базу данных (MySQL/PostgreSQL)
   - Добавить API для проверки лицензий
   - Настроить SSL сертификат
   - Добавить WAF (Web Application Firewall)

## 🐛 Известные ограничения:

1. **localStorage:**
   - Данные хранятся локально в браузере
   - При очистке кэша данные удаляются
   - Для production нужна база данных

2. **Защита:**
   - Защита работает на клиенте
   - Опытный хакер может обойти
   - Для полной защиты нужен backend

3. **Платежи:**
   - Нет автоматической интеграции
   - Нужно вручную выдавать ключи
   - Можно добавить Telegram бота

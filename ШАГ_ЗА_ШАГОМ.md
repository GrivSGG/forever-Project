# 📋 Пошаговая инструкция настройки Firebase

## 🎯 ШАГ 1: Настройка правил Firestore

### Вариант А: Прямая ссылка
1. Откройте эту ссылку в браузере:
   ```
   https://console.firebase.google.com/project/forever-client/firestore/rules
   ```

### Вариант Б: Через консоль Firebase
1. Откройте: https://console.firebase.google.com
2. Выберите проект **forever-client**
3. В левом меню нажмите **Firestore Database**
4. Нажмите на вкладку **Rules** (Правила)

### Что делать дальше:
1. Вы увидите редактор с текущими правилами
2. Удалите ВСЁ что там есть
3. Скопируйте и вставьте эти правила:

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /users/{userId} {
      allow read, write: if true;
    }
    match /tickets/{ticketId} {
      allow read, write: if true;
    }
    match /licenses/{licenseId} {
      allow read, write: if true;
    }
    match /bans/{banId} {
      allow read, write: if true;
    }
  }
}
```

4. Нажмите кнопку **Publish** (Опубликовать) справа вверху
5. Подтвердите публикацию

✅ Правила настроены!

## 🔧 Если что-то не работает

### Проблема: Пользователь не появляется в панели хелпера

**Решение:**
1. Откройте консоль браузера (F12)
2. Проверьте ошибки
3. Убедитесь что правила Firestore опубликованы
4. Сделайте жесткую перезагрузку (Ctrl+Shift+R)

### Проблема: Ошибка "Permission denied"

**Решение:**
1. Правила Firestore не опубликованы
2. Вернитесь к ШАГу 1
3. Убедитесь что нажали **Publish**

### Проблема: Старая версия сайта

**Решение:**
1. Подождите 2-3 минуты
2. Сделайте жесткую перезагрузку (Ctrl+Shift+R)
3. Очистите кэш браузера

### Проблема: Firebase не инициализирован

**Решение:**
1. Проверьте что файлы загружены на GitHub
2. Проверьте что `firebase-config.js` существует
3. Откройте консоль (F12) и проверьте ошибки

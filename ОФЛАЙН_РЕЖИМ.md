# 📵 Работа в офлайн режиме

## Как работает синхронизация офлайн?

### ✅ Что работает БЕЗ интернета:

1. **Регистрация** - данные сохраняются в localStorage
2. **Создание тикетов** - сохраняются локально
3. **Баны пользователей** - сохраняются локально
4. **Генерация ключей** - сохраняются локально
5. **Просмотр данных** - все локальные данные доступны

### 🔄 Автоматическая синхронизация при восстановлении интернета:

Система **автоматически определяет** когда интернет восстанавливается и **немедленно синхронизирует** все изменения!

```
1. Вы офлайн → Регистрируете пользователя
2. Данные сохраняются в localStorage
3. Интернет восстанавливается
4. Система автоматически обнаруживает это
5. Через 1 секунду запускается синхронизация
6. Данные отправляются в GitHub Gist
7. Все устройства получают обновления
```

## 🎯 Как это работает технически:

### 1. Обнаружение офлайн режима

При загрузке страницы система проверяет статус интернета:
```javascript
if (!navigator.onLine) {
    console.log('📵 Вы офлайн. Синхронизация будет выполнена при подключении.');
}
```

### 2. Обнаружение восстановления интернета

Когда интернет восстанавливается:
```javascript
window.addEventListener('online', () => {
    console.log('🌐 Интернет восстановлен! Запуск синхронизации...');
    window.syncSystem.forceSync();
});
```

### 3. Локальное хранение

Все данные всегда сохраняются в localStorage:
- `forever_users` - пользователи
- `forever_tickets` - тикеты
- `bans` - баны
- `keyHistory` - ключи

## 📋 Сценарии использования:

### Сценарий 1: Регистрация офлайн

1. Пользователь открывает сайт без интернета
2. Регистрируется
3. Данные сохраняются в localStorage
4. Видит сообщение: "Регистрация успешна!"
5. Когда появится интернет - данные автоматически синхронизируются

### Сценарий 2: Создание тикета офлайн

1. Пользователь создает тикет без интернета
2. Тикет сохраняется локально
3. Видит сообщение: "Тикет создан!"
4. При восстановлении интернета - тикет отправляется в облако
5. Helper видит тикет на своем устройстве

### Сценарий 3: Helper работает офлайн

1. Helper открывает панель без интернета
2. Видит все локальные данные
3. Может банить пользователей, генерировать ключи
4. Все изменения сохраняются локально
5. При подключении - все синхронизируется

## 🔍 Проверка статуса:

### В консоли (F12):

```javascript
// Проверить онлайн/офлайн
navigator.onLine  // true = онлайн, false = офлайн

// Статус синхронизации
window.syncSystem.getStatus()

// Принудительная синхронизация (если онлайн)
await window.syncSystem.forceSync()
```

## ⚠️ Важные замечания:

### Что НЕ работает офлайн:

1. ❌ **Синхронизация между устройствами** - требует интернет
2. ❌ **Отправка в GitHub Gist** - требует интернет
3. ❌ **Получение данных с других устройств** - требует интернет

### Что ВСЕГДА работает:

1. ✅ **Локальное сохранение** - работает всегда
2. ✅ **Просмотр локальных данных** - работает всегда
3. ✅ **Создание/изменение данных** - работает всегда
4. ✅ **Автосинхронизация при восстановлении** - работает автоматически

# Настройка синхронизации через GitHub Gist API

## Что это дает?
Все регистрации, тикеты, баны и ключи будут автоматически синхронизироваться между всеми устройствами через GitHub Gist. Теперь если пользователь зарегистрируется с телефона, вы увидите его в панели хелпера на компьютере!

## Как настроить?

### Шаг 1: Создать GitHub Personal Access Token

1. Зайдите на GitHub: https://github.com/settings/tokens
2. Нажмите "Generate new token" → "Generate new token (classic)"
3. Дайте название токену: `Forever Client Sync`
4. Выберите срок действия: `No expiration` (без срока)
5. Отметьте галочку: `gist` (Create gists)
6. Нажмите "Generate token"
7. **ВАЖНО**: Скопируйте токен! Он показывается только один раз!

Токен будет выглядеть примерно так: `ghp_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx`

### Шаг 2: Настроить синхронизацию

Откройте консоль браузера (F12) на любой странице сайта и выполните:

```javascript
await window.syncSystem.configure('ВАШ_ТОКЕН_ЗДЕСЬ');
```

Например:
```javascript
await window.syncSystem.configure('ghp_1234567890abcdefghijklmnopqrstuvwxyz');
```

### Шаг 3: Проверить статус

Проверьте, что синхронизация работает:

```javascript
window.syncSystem.getStatus();
```

Должно показать:
```javascript
{
  enabled: true,
  configured: true,
  lastSync: 1234567890,
  inProgress: false
}
```

## Как это работает?

1. **Автоматическая синхронизация**: Каждые 30 секунд данные автоматически синхронизируются
2. **Принудительная синхронизация**: После регистрации/создания тикета автоматически запускается синхронизация
3. **Слияние данных**: Данные с разных устройств объединяются без дубликатов
4. **Безопасность**: Gist приватный, доступен только вам

## Ручная синхронизация

Если нужно синхронизировать данные прямо сейчас:

```javascript
await window.syncSystem.forceSync();
```

## Проверка данных в Gist

1. Зайдите на https://gist.github.com/
2. Найдите Gist с названием "Forever Client - Data Sync"
3. Там будет файл `forever-client-data.json` со всеми данными

## Отключение синхронизации

Если нужно временно отключить:

```javascript
window.syncSystem.stopSync();
```

Включить обратно:

```javascript
window.syncSystem.startSync();
```

## Важные замечания

- Токен сохраняется в localStorage браузера
- Gist ID также сохраняется локально
- При первой настройке автоматически создается новый Gist
- Если очистить localStorage, нужно будет настроить заново
- Синхронизация работает на всех страницах: главная, dashboard, helper panel

## Устранение проблем

### Синхронизация не работает

1. Проверьте токен:
```javascript
localStorage.getItem('github_token');
```

2. Проверьте Gist ID:
```javascript
localStorage.getItem('sync_gist_id');
```

3. Проверьте консоль на ошибки

### Сбросить настройки

```javascript
localStorage.removeItem('github_token');
localStorage.removeItem('sync_gist_id');
location.reload();
```

Затем настройте заново с Шага 2.

## Безопасность

- Никогда не делитесь своим GitHub токеном
- Не коммитьте токен в репозиторий
- Gist автоматически создается как приватный
- Только вы можете видеть данные в Gist

## Поддержка

Если возникли проблемы, проверьте консоль браузера (F12) на наличие ошибок. Все операции синхронизации логируются с эмодзи:
- 🔄 - Начало синхронизации
- ✅ - Успешная синхронизация
- ❌ - Ошибка синхронизации
- 📢 - Данные обновлены
